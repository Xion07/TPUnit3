FACTURAZION 3
1.  En lugar de tener todo el código en el método main, se dividió en métodos más pequeños, cada uno con una responsabilidad específica.

3. En lugar de escribir directamente los valores de las fechas, se utilizó constantes para mejorar la legibilidad y facilitar los posibles cambios futuros.

4. Se crea clases separadas para el código (Factura), quedando así imprimirFactura y la facturacionM3. Esto permitirá una mayor modularidad y reutilización del código.

5. En lugar de utilizar la clase List para representar la lista de precios en la factura, se utiliza la interfaz List. Esto permitirá que el código sea más flexible.

6. Se utiliza la anotación @Override para indicar que estás reemplazando una implementación existente.
